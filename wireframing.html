Brief & process

Wireframing Project Buddhify website

The wireframing phase was divided into two parts: the first one showing desktop wireframe and the second one showing the mobile version

Process Part 1

  1. Summary of opportunities and issues identified from the user journey analysis
  2. Create five mid-fidelity wireframes incorporating improvements to the website
  3. Develop a wireflow to visualize user interactions
  4. Reorganize the site’s information architecture to enhance usability

Wireframing Project Part 1

5 redesigned wireframe pages for the Buddhify website

I conducted an analysis of five pages of the Buddhify website and recreated their wireframes, integrating the improvements identified during the research.

Each wireframe is accompanied by a detailed explanation outlining the design decisions and enhancements made.

Wireframing Project Part 2

Mobile version

I redesigned the mobile wireframes for the Buddhify site were designed to maintain consistency with the desktop version, focusing on clarity, accessibility, and intuitive navigation.

Key features include touch-friendly buttons, a hamburger menu and search icon in the header, and an expandable footer for organized content.
